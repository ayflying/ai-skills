# 微信机器人快速测试指南

## 测试前准备

### 1. 风险提示
- ⚠️ 个人微信自动化可能违反微信使用条款
- ⚠️ 可能导致账号被限制或封禁
- ⚠️ 网页版微信功能有限，部分功能不可用

### 2. 建议
- ⚠️ **务必使用测试专用微信号**
- ⚠️ 不要使用主力账号或重要账号
- ⚠️ 了解可能存在的风险

## 测试步骤

### 步骤1：基础测试（无需微信账号）
```bash
# 进入技能目录
cd skills/wechat-bot

# 运行基础测试
python scripts/test_bot.py
```
**预期结果**：测试结果: 3/3 通过

### 步骤2：配置检查
```bash
# 查看配置文件
cat .env

# 以测试模式运行
python scripts/wechat_bot.py --test
```
**预期结果**：显示配置信息，不显示二维码

### 步骤3：实际运行测试（需要微信账号）

#### 准备工作
1. 准备一个**专门的测试微信号**
2. 确保该账号可以正常登录网页版微信
3. 确保网络连接正常

#### 启动机器人
```bash
# 启动机器人
python scripts/wechat_bot.py
```

#### 登录步骤
1. 运行命令后会显示二维码
2. 使用**测试微信账号**扫描二维码
3. 在手机上确认登录

### 步骤4：功能测试

#### A. 消息收发测试
1. 用另一个微信账号向机器人发送文本消息
2. 检查机器人是否收到消息
3. 检查是否收到自动回复（如果启用）

#### B. 命令测试
发送以下命令测试：
```
/help    - 显示帮助信息
/status  - 显示机器人状态
/stats   - 显示消息统计
/export  - 导出收集的数据
```

#### C. 媒体文件测试
1. 发送图片消息
2. 发送语音消息
3. 发送视频消息
4. 发送文件
5. 检查 `data/media/` 目录是否保存

#### D. 数据检查
```bash
# 查看消息记录
ls -la data/messages_*.json

# 查看日志文件
ls -la logs/*.log
```

### 步骤5：测试完成清理
1. 按 `Ctrl+C` 停止机器人
2. 在手机微信上退出网页版登录
3. 检查并备份重要数据
4. 可选：删除测试数据
```bash
# 清理测试数据
rm -rf data/ logs/  # Linux/Mac
rmdir /s /q data logs  # Windows
```

## 故障排除

### 常见问题

| 问题 | 解决方案 |
|------|----------|
| 无法导入wxpy | `pip install wxpy python-dotenv` |
| 二维码不显示 | 尝试其他终端或使用`console_qr=False` |
| 登录失败 | 使用测试账号，检查网络连接 |
| 消息收不到 | 重新启动机器人 |
| 发送失败 | 检查联系人名称 |

### 调试模式
```bash
# 启用详细日志
export LOG_LEVEL=DEBUG
python scripts/wechat_bot.py
```
